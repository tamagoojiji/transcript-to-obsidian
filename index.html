文字起こし → Obsidian 自動仕訳ツール

Claude Code で文字起こしファイルを自動でObsidianノートに変換

このツールでできること

  • フォルダに入れた文字起こしファイルを1本ずつ自動で処理
  • AIが内容を読み、要約・キーポイント抽出・テーマ分類
  • Obsidianのフォルダに整理されたMarkdownノートとして保存
  • 処理済みファイルは自動でスキップ（途中再開OK）
ポイント: フォルダ名を2箇所変えるだけ。あとは実行して放置するだけです。

対応ファイル形式

形式拡張子説明
テキスト.txtプレーンテキスト
字幕.srt .vttタイムスタンプ付き字幕
CSV.csv表形式の文字起こし
JSON.jsonAPI出力等
Markdown.mdMarkdown形式

必要なもの

  • Claude Code（インストール済み前提）
  • Obsidian（保存先として使用）

ファイル構成

transcript-to-obsidian/
├── process.sh ← メインスクリプト（実行するファイル）
├── config.sh ← 設定ファイル（ここだけ編集）
└── prompt-template.md ← AI への指示（カスタマイズ可）

セットアップ手順

1ツールをダウンロード

git clone https://github.com/tamagoojiji/transcript-to-obsidian.git
cd transcript-to-obsidian

2config.sh を編集（2箇所だけ）

テキストエディタで config.sh を開いて、フォルダのパスを変更:

# 文字起こしファイルが入っているフォルダ（絶対パス）
INPUT_DIR="$HOME/transcripts"

# Obsidianの保存先フォルダ（絶対パス）
OUTPUT_DIR="$HOME/My-Knowledge/voicy"
INPUT_DIR 文字起こしファイルが入っているフォルダのパス
OUTPUT_DIR ObsidianのVaultフォルダ内の保存先パス
パスの確認方法: Finderでフォルダを右クリック →「パス名をコピー」で正確なパスが取得できます。 $HOME はホームフォルダ（/Users/あなたの名前）の意味です。

3実行

./process.sh

あとは放置するだけ。進捗がターミナルに表示されます。

仕訳ルールをカスタマイズしたい場合

prompt-template.md を編集すると、AIへの指示を変更できます。

デフォルトでは以下の形式で出力されます:

---
source: episode001.txt
date: 2026-04-16
---

# エピソードタイトル

## 要約
3〜5行の要約...

## キーポイント
- ポイント1
- ポイント2
- ...

## 詳細メモ

### テーマ1
内容...

### テーマ2
内容...

#タグ1 #タグ2 #タグ3

例えば、以下のようなカスタマイズが可能です:

  • 出力を英語にする
  • 特定のテーマだけ抽出する
  • タグの体系を指定する
  • 名言・格言だけ抜き出す

コストの目安

プラン 追加コスト 処理速度 注意点
Max $100 おすすめ $0（定額内） 70本: 約1〜2時間 レート制限あり（自動待機）
Pro $20 $0（定額内） 70本: 約2〜4時間 レート制限が厳しめ
API従量課金 約$15（Sonnet） 70本: 約40分 レート制限ほぼなし
モデルについて: デフォルトは sonnet（高速・低コスト）です。 文字起こしの仕訳・要約にはSonnetで十分な品質が得られます。 より高度な分析が必要な場合は config.sh で opus に変更できます。

よくある質問

途中で止まった場合は？

もう一度 ./process.sh を実行するだけでOK。処理済みファイルは自動でスキップされます。

「Permission denied」と出る場合

chmod +x process.sh

レート制限で待機が多い場合

config.sh の WAIT_SECONDS を大きくしてください（デフォルト: 30秒）。

仕訳の精度を上げたい場合

prompt-template.md に具体的な指示を追加してください。 例: 「ビジネス/マインドセット/習慣 の3カテゴリに分類」など。
